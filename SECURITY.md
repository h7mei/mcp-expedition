# Security Policy

## Supported versions

This project is currently in early development (`0.x`). Security fixes are applied to the default branch.

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Email the maintainers with:

- A description of the issue
- Steps to reproduce
- Impact assessment
- Any suggested fix

We will acknowledge reports as quickly as practical and coordinate disclosure.

## Safe usage notes

- Only point the Workspace server at trusted directories
- Treat write tools as approval-gated
- Do not place secrets in fixture workspaces
- Review protocol traces before sharing them
