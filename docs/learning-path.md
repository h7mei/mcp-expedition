# Learning path

1. **Foundation** — monorepo, schemas, config, lint/test/CI
2. **Local server** — Workspace MCP server over `stdio`
3. **Custom client** — reusable MCP client + CLI host
4. **Model integration** — sampling and prompt-driven workflows
5. **Client capabilities** — roots, elicitation, progress, logging
6. **Remote transport** — Streamable HTTP ResearchOps server
7. **Authorization** — authn/authz for remote surfaces
8. **Testing and debugging** — Inspector, traces, interoperability suites
9. **Registry publication** — server metadata and release process
10. **Specification study and contribution** — deeper MCP protocol work
